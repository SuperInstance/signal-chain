The Problem

✕

Pure Algorithms

Fast, deterministic, testable. Shatter on novel inputs. Your system is as smart as your last deploy.

VS
✕

Pure Models

Flexible, creative, expensive. Re-derive what code already knows. Your system is as smart as your budget.

Both treat the entire pipeline as a single unit. But real pipelines have stages — and each stage has different needs.

The Insight

The missing parameter: a per-room dial

Filterα = 0.0
Validateα = 0.2
Classifyα = 0.4
Reasonα = 0.6
Escalateα = 0.8
Agentα = 1.0

α ∈ [0, 1]  where 0 = pure code, 1 = full agent, and the interesting territory is in between

The Metaphor

A guitarist doesn't plug into an amp. They build a signal chain.

🎸 Guitar
input
Raw input tile enters the chain — unprocessed data, unclassified, full potential.
→
🔊 Pedals
filter
Rule-based filtering — deterministic, fast, α ≈ 0. Catches the obvious stuff.
→
📻 Amp
classify
Micro-model shapes the signal — handles edge cases, α ≈ 0.2–0.4. The warmth pedal.
→
🔈 Speaker
deadband
Deadband detector — volume pedal. Quiet when KPIs are fine, loud when they breach.
→
🎤 Mic
reason
Medium model with tile context — complex reasoning, accumulated state, α ≈ 0.6.
→
🎛️ Console
output
Seed validation → locked pattern → output tile. Crystallized intelligence.
